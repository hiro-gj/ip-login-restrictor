# IP Login Restrictor v1.4.1 - 変更履歴

## 変更: プロキシ対応とIPv6関連の判定精度向上、および個別IP制限の誤判定修正

### 概要
プロキシ経由のアクセスを正しく判定できるように `HTTP_X_FORWARDED_FOR` を優先する処理を追加し、IPv6の判定精度を向上させました。また、フロントページ（最新の投稿一覧等）において、特定の記事の臨時IP制限が意図せずページ全体に適用されてしまう不具合を修正しました。

### 主な変更点

#### 1. プロキシ（X-Forwarded-For）対応
- 認証フックでリモート IP を取得する際に `HTTP_X_FORWARDED_FOR` が存在する場合は先頭の IP（クライアント側）を優先して使用するように変更
- これにより、ロードバランサやリバースプロキシ配下での正しいアクセス制御が可能に

#### 2. IPv6 の単一アドレス比較での正規化対応
- 単一 IP を比較する際に IPv6 の表記ゆれを統一するため `inet_pton` を適用して比較するように変更
- 例: `2001:db8::1` と `2001:0db8:0:0:0:0:0:1` を同一と判定

#### 3. IPv6 サブネット判定の実装追加・修正
- IPv6 に対するサブネット判定を実装（`inet_pton` を用いたバイト単位のマスク処理）
- プレフィックス長が 8 の倍数でない場合の残りビット処理にも対応
- /128（単一アドレス）比較の最適化

#### 4. ページ単位の臨時IP制限の誤爆修正
- フロントページが「最新の投稿」一覧などに設定されている場合、アーカイブ表示の際にたまたま先頭に位置する記事の個別IP制限（Temporary IPs）がページ全体に誤って適用されてしまう（結果として意図しないアクセス拒否が発生する）不具合を修正
- `is_singular()` の判定をフォールバック条件に追加することで、個別記事ページでのみ制限が正しく適用されるように改善

### 技術的な詳細

#### ロジックの改善・修正
- **Class_IP_Utils.php**
  - 単一アドレス比較時およびサブネット判定時に `inet_pton` を用いたバイナリレベルでの判定機能を実装
- **ip-login-restrictor.php**
  - `check_access` メソッドにおいて、フロントエンドで `$post_id` が `0` の場合（アーカイブページ等）に、グローバルの `$post->ID` を取得する処理へ `is_singular()` チェックを追加

### 互換性
- 既存のIP設定および臨時IP設定はそのまま引き継がれます。
- プロキシ環境で導入する場合、X-Forwarded-For を信頼するポリシーに注意してください（信頼できないプロキシ経由だと誤判定の原因になる場合があります）。
